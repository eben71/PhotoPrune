# Task Discovery Prompt

When a gap is found, do not expand the current task silently. Create or update a backlog entry in `docs/delivery/TASK_BACKLOG.md` with ID, title, status, priority, type, goal, links, acceptance criteria, checks, and residual risk.

Use `Draft` for unclear work and `Ready` only when acceptance criteria are actionable. Update `docs/delivery/ITERATION_LOG.md` with discovered follow-ups.
